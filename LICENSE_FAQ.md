# EllipticZero License FAQ

This FAQ is a practical reading guide for the repository license. It is not a
replacement for the `LICENSE` file. If there is any conflict, the `LICENSE` file
controls.

## What License Does EllipticZero Use?

EllipticZero is distributed under `FSL-1.1-ALv2`.

That means the current repository versions are source-available for permitted
purposes, while certain commercial product, hosted-service, or competing-use
scenarios should be handled through a separate agreement.
Each published version later becomes available under Apache License 2.0 on the
second anniversary of the date that version was made available.

## Can I Evaluate The Project?

Yes. The public repository is intended to be readable and evaluable. You can
review the source, run local checks, test the CLI in `mock` mode, and evaluate
the full agent workflow with your own configured providers and API keys under
the public license terms.

`mock` mode is only the easiest no-key starting point. It is not the only
evaluation path.

## Can I Use It Internally?

The public license includes internal use as a permitted purpose, subject to the
actual license terms. For sensitive deployment or commercial-product questions,
contact before rollout so the intended use can be clarified.

## When Should I Ask About A Commercial License?

Contact before shipping, selling, or deploying if your plan involves one of
these scenarios:

- competing commercial use
- hosted or managed service use
- SaaS or platform use based on EllipticZero
- OEM or embedded commercial distribution
- white-label use
- resale
- commercial security platform integration where EllipticZero or substantially
  similar functionality is part of a paid product or service

## Does FSL Protect The Idea?

No license protects an abstract idea by itself. FSL applies to the software
made available under the license. The commercial position is strengthened by the
current codebase, artifacts, roadmap, documentation, brand, release history,
support, and private commercial terms.

## What Happens After Two Years?

Each published version becomes available under Apache License 2.0 on the second
anniversary of the date that version was made available. Newer versions can
still be published under FSL with their own future-license date.

## Is This Open Source?

Current versions are public and source-available, but they should not be
described as OSI-approved open source. Use `source-available` when describing
the current repository versions.
